# AI Design Prompt Library

An open-source collection of professional prompt frameworks for designers, art directors, and creative technologists working with generative AI.

The library focuses on **creative direction, visual systems, realism, composition, typography, and production quality** rather than generic one-click prompts.

## Why this project exists

AI can execute design instructions, but strong results still require human taste, direction, references, and judgment.

This project helps creators translate visual ideas into structured instructions while keeping the creative decision-making human-led.

## How to use the library

1. Choose the visual direction closest to your project.
2. Open the relevant prompt collection.
3. Replace bracketed placeholders such as `[PRODUCT]`, `[BRAND]`, and `[MESSAGE]`.
4. Add the intended format, audience, and campaign objective.
5. Generate an initial result.
6. Review hierarchy, anatomy, typography, materials, lighting, and brand consistency.
7. Rewrite the weakest part instead of repeatedly generating the same prompt.

### Basic example

```text
Create a [FORMAT] campaign visual for [BRAND OR PRODUCT].

Creative direction:
[DESCRIBE THE CENTRAL VISUAL IDEA AND EMOTIONAL INTENT]

Composition:
[DEFINE SUBJECT PLACEMENT, CAMERA ANGLE, DEPTH, AND NEGATIVE SPACE]

Art direction:
[DEFINE TYPOGRAPHY, MATERIALS, COLOR SYSTEM, AND GRAPHIC ELEMENTS]

Lighting:
[DEFINE LIGHT SOURCE, CONTRAST, REFLECTIONS, AND SHADOW QUALITY]

Quality controls:
Natural materials, intentional hierarchy, realistic anatomy, coherent reflections,
accurate perspective, no generic stock-image styling, and no unnecessary visual clutter.
```

## How to contribute

Contributions are welcome from designers, developers, researchers, and creative technologists.

Useful contributions include:

- New prompt frameworks
- Improvements to existing prompts
- Model-specific guidance
- Better quality-control instructions
- Documentation improvements
- Before-and-after examples
- New visual directions

Before submitting a pull request, read [CONTRIBUTING.md](CONTRIBUTING.md).

A typical contribution workflow is:

1. Open an issue describing the proposed improvement.
2. Create a focused branch.
3. Make one clear change.
4. Open a pull request.
5. Explain what changed and why.
6. Link the issue using `Closes #issue-number`.

## Project principles

- Human direction comes first.
- Prompts should communicate intent, not only aesthetics.
- Strong hierarchy is more valuable than decorative complexity.
- Avoid copying living designers or active campaigns too closely.
- Never publish private client material.
- Credit contributors and external references.
- Do not submit prompts intended for deception, impersonation, or harmful content.
